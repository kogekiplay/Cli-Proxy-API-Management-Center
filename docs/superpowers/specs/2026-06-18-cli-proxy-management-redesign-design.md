# CLI Proxy API 管理中心 · 重新设计（Claude 风格视觉原型）

- **日期**：2026-06-18
- **状态**：视觉方向已确认，待实现计划
- **作者**：设计 brainstorm 会话产出
- **参考**：现有项目 `Cli-Proxy-API-Management-Center`（React 19 + TS 6 + Vite 8 + SCSS Modules），Go 后端 `CLIProxyAPI`

## 1. 背景与动机

现有管理页是一个数据密集型后台（10 个页面、表格、表单、YAML 编辑器）。其设计采用「极简暖灰」调性，但**主行动色被设成接近灰色**（`#8b8680`），导致强调色失效、界面扁平、缺乏层次与呼吸感——读起来「丑」且低能量。

用户希望基于此项目重新做一个**更好看**的项目，方向确定为 **Claude 风格**：暖奶油/纸感底色、克制的黏土/珊瑚色强调、编辑感衬线标题 + 干净无衬线正文、慷慨留白、柔和景深。本质不是发明新身份，而是**把 Claude 的视觉语言做对**。

## 2. 范围

**视觉原型 + mock 数据，先做核心页**。新建一个独立项目（不直接改原项目），用 mock 数据把四个核心页面的视觉与交互做到位。用户满意后再扩展到全部 10 个页面并接入真实 Management API。

四个原型页面：
1. **登录页**（独立、无外壳，第一印象）
2. **仪表盘**（同时定义整个 App 外壳：左侧导航 + 顶栏 + 内容区）
3. **AI 提供商**（最复杂的交互工作台）
4. **认证文件**（列表 / 筛选 / 上传 / 批量操作类数据管理页）

不含：配额、OAuth、插件、配置、日志、系统信息页；真实 API 接入；国际化（i18n）；单文件打包为 `management.html`。这些留待后续扩展阶段。

## 3. 技术栈

- **React 19 + Vite + Tailwind v4**
  - 选 Tailwind v4 的 CSS-first 设计 token 机制，最适合落地这套高度定制的 Claude 设计系统（自定义调色板、字体、圆角、阴影）。
  - 留在 React 生态，后续若做完整功能版，可把现有项目的 API 层 / Zustand store / i18n 逻辑移植进来。
- **状态**：原型阶段用本地 mock 数据 + React state；不引入 Zustand（页面少、无跨页共享状态需求）。
- **路由**：`react-router-dom` v7（HashRouter，与原项目一致，便于将来对齐）。
- **图标**：内联 SVG 组件（与原型 mockup 中一致的线框风格）。
- **字体**：Google Fonts —— Source Serif 4（衬线标题）、Inter（UI/正文）、JetBrains Mono（代码/YAML/文件名）。

> 单文件打包约束暂不施加。原型是独立项目正常构建；将来扩展到完整功能版时，再用 `vite-plugin-singlefile` 打成可替换 `management.html` 的单文件。

## 4. 视觉设计系统（Design Tokens）

### 4.1 配色（浅色主题，默认）

**表面 / Surfaces — 暖奶油纸感**
| Token | 值 | 用途 |
|---|---|---|
| `--paper` | `#EFEBE1` | 页面底色 |
| `--paper-deep` | `#E8E3D6` | 次级底 / 分段背景 |
| `--surface` | `#FAF9F5` | 侧栏等次级面 |
| `--surface-raised` | `#FFFFFF` | 卡片 / 浮起面 |
| `--line` | `#E4DFD3` | 边框 / 分割线 |
| `--line-strong` | `#D6D0C1` | 强边框 / 输入框 |

**文字 / Ink — 暖近黑**
| Token | 值 | 用途 |
|---|---|---|
| `--ink` | `#262320` | 主文字 |
| `--ink-2` | `#6B675E` | 次要文字 |
| `--ink-3` | `#9A958A` | 弱化 / 占位 |

**强调 + 语义**
| Token | 值 | 用途 |
|---|---|---|
| `--clay` | `#C96442` | 主行动色（黏土赤陶） |
| `--clay-hover` | `#B5512F` | hover |
| `--clay-tint` | `rgba(201,100,66,.10)` | 强调底色 |
| `--success` | `#5E8C6A` | 成功 / 健康 |
| `--gold` | `#BF8B2E` | 提示 / 配额偏高 |
| `--danger` | `#B5483B` | 危险 / 删除 |

语义色全部压暖，与纸感底色协调，不出现冷蓝/冷绿。

### 4.2 字体

- **衬线 Display**：Source Serif 4，weight 600，用于页面/区块大标题、统计数字。
- **无衬线 UI**：Inter，400/500/600/700，正文与界面。
- **等宽**：JetBrains Mono，用于代码、YAML、认证文件名、Base URL。

### 4.3 圆角 / 阴影 / 间距

- 圆角：`--radius-sm 9px` / `--radius 14px` / `--radius-lg 20-22px`。药丸=9999px。
- 阴影：`--shadow-sm`（卡片默认）/ `--shadow-md`（hover 浮起）/ `--shadow-lg`（模态/批量栏）。
- 间距以 4/8/12/16/22/30 为节奏；强调「呼吸感」，容器内边距偏大（卡片 16-22px，面板 22-30px）。

### 4.4 暗色主题

原型阶段以浅色为主，但 token 结构预留暗色映射（纸感深棕底 `#151412`/`#1d1b18`，黏土色提亮为 `#9a948e`→维持 `#C96442` 作 accent）。三主题切换（浅 / 纸白 / 暗）在侧栏底部有控件位，原型可仅浅色可点。

## 5. App 外壳（Shell）

仪表盘页同时确立外壳，所有内页复用：

- **左侧导航（256px）**：
  - 顶部品牌标（黏土渐变方块 + 衬线「CLI Proxy / 管理中心」）。
  - 分组导航：**总览**（仪表盘 / AI 提供商 / 认证文件 / OAuth / 配额管理）、**系统**（配置面板 / 日志 / 系统信息）。
  - 当前项：黏土色淡底 + 左侧 3px 黏土竖条 + 加粗；带计数徽章。
  - 底部：连接状态卡（脉动绿点 + host + 「管理密钥 · 已连接」）+ 主题切换分段控件。
- **顶栏**：衬线大标题 + 面包屑副标题；右侧操作区（状态药丸 + 图标按钮 + 主操作按钮）；半透明毛玻璃背景。
- **内容区**：可滚动，统一内边距。

## 6. 页面设计

### 6.1 登录页

- 全屏两栏布局（窄屏自动单栏）。
- **左栏 编辑式面板**：品牌标 + 衬线大标题（「用一个安静、克制的界面 *管理* 你的代理服务」，"管理"斜体黏土色）+ 副标题。背景暖色光晕。
  - **不设功能要点列表**（已移除）。
  - 底部：语言选择 + 「≥ v7.1.0 推荐」。
- **右栏 表单卡**：「连接到服务」标题；API 地址、管理密钥两个带图标输入框；「自动推断地址」开关 + 「清除本地登录信息」；黏土色实心「连接」按钮。
  - 分隔线下：`/management.html` 说明卡 + 安全提示卡（密钥存 localStorage、远程需 `allow-remote-management`）。

### 6.2 仪表盘

- **四个统计磁贴**：在线提供商 / 认证凭据 / 代理 API Keys / 可用模型。衬线数字 + 暖色图标 + 趋势小字。
- **双栏**：服务信息（版本/构建/运行时长/Management API 状态，带金色「有新版本」提示）+ 可用模型（按提供商分组的等宽小标签）。
- **提供商健康度**：每行 logo 徽标 + 名称 + 凭据数 + 配额进度条（91% 自动转金色预警）。

### 6.3 AI 提供商（工作台）

- **左栏提供商列表**：分「OAuth 提供商」「API 提供商」两组，每项 logo 徽标 + 凭据数 + 状态点（绿=正常、金=配额偏高、灰=未配置）。
- **右栏详情**（关键：两类提供商语义不同，已据真实 `descriptors.ts` 校正）：
  - **OAuth 提供商**（Claude / Gemini CLI / Codex / Antigravity / Kimi / xAI）：无上游 Base URL（请求经 CPA 自身转发），无模型别名映射，只有 OAuth 获取的模型列表 + 排除模型；主操作 = **「通过 XXX 登录」** 或 **「上传认证文件」**。
  - **API 提供商**（OpenAI 兼容 / Vertex，及 config 里配 key 的 Gemini/Codex/Claude）：有 Base URL、API Keys、模型别名、排除模型、Prefix、Headers；主操作 = 测试连接 / 添加 Key。
  - 详情头卡：大 logo + 名称（衬线）+ 类型 + 操作按钮 + 元信息行。
  - 标签页：认证文件 / 模型 / 排除模型 / 设置。
  - 认证文件表格：头像缩写 + 邮箱/标识 + 文件名，状态药丸，可用模型数，最近使用，行内图标操作。

### 6.4 认证文件

- **筛选标签行**：按提供商类型（全部/Claude/Gemini CLI/Codex/Antigravity/OpenAI），带头像 + 计数，选中态实心黏土。
- **控制行**：搜索（支持 `*` 通配）、每页数量、排序、「仅问题项 / 仅禁用 / 紧凑」开关。
- **卡片网格**（3 列），覆盖真实状态：
  - 已选中态（黏土描边 + 光晕）、配额偏高（金色）、带最近请求 sparkline、已禁用（整卡变暗 + 开关关）、虚拟/仅内存（虚线框、无勾选/删除）。
  - 每卡：类型徽章 + 状态徽章 + 文件名（等宽）+ 元信息（大小/修改/优先级）+ 成功/失败统计 + 配额条 + 操作（模型/下载/Prefix&代理设置/删除 + 启用禁用开关）。
- **分页**：上一页 / 第 X/Y 页 · 共 N 份 / 下一页。
- **底部两张卡**：OAuth 排除模型（通配标签）+ OAuth 模型别名（`from → to` 映射列表）。
- **批量操作栏**：选中后底部居中浮现（深色浮层），含全选/反选/下载/启用/禁用/删除。

## 9. 不做（YAGNI）

- 真实 Management API 接入、Zustand store、i18n、单文件打包——留给扩展阶段。
- 配额 / OAuth / 插件 / 配置 / 日志 / 系统信息页——留给扩展阶段。
- 暗色主题完整实现——仅 token 预留。
- 动画库（Motion）——原型用 CSS transition 即可。

## 10. 验收标准

- 四个页面用 mock 数据可独立浏览、视觉与本文档一致。
- 设计 token 集中定义，组件可复用。
- `bun run dev`（或 `npm run dev`）可在本地启动预览。
- 响应式：窄屏登录页单栏；内页侧栏可收起（原型至少不破版）。
