# 请求监控模型别名与状态提示设计

## 目标

- OpenAI 兼容提供商在监控表格中显示配置名称，例如 `openai-compatible-cf worker` 显示为 `cf worker`。
- 配置了模型 alias 时，监控、筛选、聚合和费用计算统一使用 alias。
- 已有历史请求也按当前配置映射 alias，不要求重写历史 SQLite 数据。
- 删除独立错误信息列，将失败详情放到状态码徽章的悬停和键盘聚焦提示中。
- 缩小认证列与状态码之间的空白，同时给模型、Token 和费用保留足够空间。

## 后端数据模型

用量账本继续保存真实上游模型，并新增可选的 `model_alias`：

- `model`：实际发送给上游的模型名。
- `model_alias`：客户端请求的模型名，由运行时已有的 `usage.Record.Alias` 提供。

SQLite 初始化通过 `ALTER TABLE` 兼容已有数据库，不修改或清空旧记录。新请求写入两者；旧记录的 `model_alias` 为空。

管理接口查询时生成“有效模型名”：

1. 请求记录自身的 `model_alias` 非空时使用它。
2. 否则按当前 CPA 配置，以 provider、认证索引和上游模型名查找 alias。
3. 未匹配时回退到原始 `model`。

有效模型名用于模型筛选、模型聚合和价格查询。原始上游模型仍随事件详情返回，便于排障。这样新旧记录行为一致，同时避免不可逆地改写历史数据。

## Alias 映射

管理处理器从当前配置构建只读映射，并随本次分析请求传入账本查询：

- OpenAI-compatible 配置使用 `openai-compatible-<配置名称>` 匹配 provider。
- 优先使用认证索引限定映射，避免同一 provider 下不同凭证映射冲突。
- 无认证索引的历史记录可回退到 provider 级唯一映射。
- 映射冲突时不猜测，保留原始模型名。

模型筛选同时接受 alias 和原始模型名；返回结果统一展示有效模型名。

## 费用计算

每条分析事件先解析有效模型名，再调用现有 `CostForUsage`。因此价格表中配置 alias 的价格即可用于对应请求，不再用上游的 `@cf/...` 名称误报“未配置价格”。

汇总、时间趋势、模型统计、API Key 统计、认证文件统计和事件列表都基于同一批已计价事件，避免列表有价格但汇总仍为 `-`。

## 前端显示

请求监控表格从 12 列改为 11 列：

`时间 / 请求 / 提供商 / 模型 / 思考强度 / 认证或 API Key / 状态码 / 延迟 / Token / 费用 / 操作`

列宽比例为：

`8 / 12 / 8 / 10 / 6 / 10 / 6 / 12 / 15 / 9 / 4`

- 提供商表格标签移除 `openai-compatible-` 前缀，完整内部 ID 放在 `title` 中。
- 详情抽屉保留完整 provider ID。
- 模型主文本显示 alias；原始上游模型不同的时候在详情中显示。
- 表格最小宽度从 1440px 调整为 1320px，避免删除一列后仍显得松散。

## 错误提示

- 删除“错误信息”表头和单元格。
- `StatusBadge` 自己承载错误提示。
- 成功请求仅显示 `200`，不渲染空提示。
- 失败请求在悬停或键盘聚焦状态徽章时显示错误标题和完整详情。
- 提示框限制宽高、允许换行和滚动，不能撑开表格或被单元格裁切。
- 详情抽屉继续保留完整错误信息和复制能力。

## 测试

后端测试覆盖：

- 新请求把 `usage.Record.Alias` 写入 `model_alias`。
- 旧数据库自动新增列且原数据可读。
- 记录 alias 优先于配置映射。
- 历史记录可按当前 OpenAI-compatible 配置映射 alias。
- 映射冲突回退原模型。
- alias 价格用于事件与全部聚合费用。
- alias 和原始模型都可用于筛选。

前端测试覆盖：

- 11 列宽度总和为 100。
- OpenAI-compatible provider 表格标签正确缩短。
- 表格不再渲染错误信息列。
- 失败状态徽章包含可访问的错误提示，成功状态不包含空提示。
- 模型和上游模型在表格及详情中的显示规则。
